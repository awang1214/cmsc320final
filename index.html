Introduction¶

Valorant is a Multiplayer First Hero Person Shooter game developed by Riot Games. With over 12 million active users in 2021, Valorant has recently gotten popular over the COVID pandemic. With its rise in popularity, Riot Games has hosted tournaments involving players from all over the world.

The game is quite simple, it is a 5v5 tacitcal shooter, one side is the attackers and one side is the diefenders. The attackers are supposed to plant and detonate a spike within a map, and the defenders are to defend the site and defuse the spike if it is planted. Each game is first to 13 with 24 rounds in the main game, with a win by two condition attached. At round 12, there is a switch and attackers become defenders and defenders become attackers. If the match ends up being 12-12 overtime (2 rounds one on each side) until one team wins two rounds in a row. Players can currently play on a 7 maps, 17 agents, and several primary and secondary weapons to choose from. More information can be found here: https://playvalorant.com/en-us/news/announcements/beginners-guide/

Currently, a website called Blitz currently keeps track of data, luckily for us someone has already scraped the data and compiled it into datasets for us (/u/IronNinja) in the following repository: https://github.com/IronicNinja/valorant-stats. These statistics are a little bit outdated (July 2021) but they will suffice for the tutorial. As of this time, there are only 6 Maps and 15 Agents.

In this tutorial, our goal is to tidy the in-game data as well as provide some insight into which operators are the most successful in terms of win rate. I hope this analysis is helpful in determining what temp compositions work, as each agent is unique and serves a different function, as well as maybe help people are new to the game choose the first agent that they want to learn. Throughout this tutorial we will be coding in Python 3 and making use of many libraries (pandas, numpy, seaborn, etc.

def clean(df):
    if "Unnamed: 0" in df.columns:
        df.drop("Unnamed: 0", axis=1, inplace=True)
        
    for col in df.columns:
        df[col] = df[col].astype(str).str.rstrip('%').str.replace(',', '')
        try:
            df[col] = df[col].astype(float)
        except:
            pass  
    df.set_index(df.columns[0], inplace=True)
    return df

A quick summary of what each column + row header means:

  • row headers are all the agent names
  • KD is kill-death ratio (how many enemies killed per time dying during the period of one game)
  • Kills is how many kills the respective agent has at the end of a game
  • Deaths is how many deaths the respective agent has at the end of a game
  • Assists is how many times the agent has assisted in helping another teammate get a kill on an enemy agent per game
  • Win rate is the % the agent has won overall across the maps as of July 2021
  • Pick rate is the % the agent is picked to be played during the game
  • ACS is average combat score, which has many factors but in summary: a higher ACS is typically "better"
  • First blood is a % estimating how often the agent gets the first kill in a round
  • Num Matches is how many times the agent has been played in history. (As of July 2021)

Although from first glance it seems as though there is. a higher winrate for agents with lower ACS. But, in fact, there is something else we must consider, which is that each agent plays a different role in the game: Sentinel, controller, and duelist. This means that ACS scores might actual be biased as duelists tend to have higher ACS than controllers and sentinels. Because of this, ACS does not seem to affect the actual winrate of champions, as all of the higher ACS scores are from the duelists while all of them lower scores are results of controllers and sentinels. To find more about the roles in the game look here: https://playvalorant.com/en-us/news/announcements/beginners-guide/ As such, we should use a different parameter aside from ACS such as pickrate vs winrate. We will be doing this below.

Predicting Win Ratio¶

With the rise of popular tournaments, predicting win ratio can be helpful in determining who wins each game, there are events hosted by Riot called Pick'Ems where you can get in-game rewards and prizes if you predict the winners of matches correctly. Not only this, but third party websites allow you to bet on the winners of the games with real money as well.

In this last section we will be focusing on how to make a model to predict the winrate of a game based off of the Average combat score of the team. Although earlier I mentioned that due to the different roles average combat score varies, however, every team composition currently composes of the same amount of each role and as such should be okay to use Average combat score. Not only this, but the ACS is the most comprehensive performance indicator for in-game matches that Riot produces.

For this tutorial, we will be focusing on a linear regression model for correlation between ACS and Win-Rate, split the data into a trian and test, then use the LinearRegression to fit the data, finally use sklearn's predict function to create your predictive model.
